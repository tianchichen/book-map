# book-context-map

`book-context-map` is a writing skill for building a reusable context layer around a book note, especially in an Obsidian-based personal knowledge vault.

It is designed for the moment when a book is added to a vault and the user wants more than a summary: author background, writing context, central question, conceptual framework, reading path, controversies, and links into an existing note network.

## What It Does

This skill helps generate a "book context map" that can be inserted into or used to enrich a book note. It focuses on:

- author background
- writing context and historical setting
- the book's core question
- theoretical, methodological, or narrative framework
- core concepts
- suitable readers and use cases
- prerequisite reading
- further reading
- contrasting or critical perspectives
- links to related notes in a vault

## Best For

- nonfiction books
- theory, philosophy, business, psychology, sociology, history, biography, and methodology
- readers building long-term knowledge networks instead of one-off summaries

It can also be adapted for fiction and poetry by shifting the framework section toward themes, narrative structure, literary tradition, and character relationships.

## Output Style

The skill aims to be:

- clear
- restrained
- structured
- oriented toward knowledge reuse and later retrieval

It avoids:

- marketing copy
- encyclopedia-style biography dumps
- fake certainty
- empty summaries that simply rephrase the table of contents
