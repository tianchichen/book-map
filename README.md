# book-context-map

[中文说明](./README.zh-CN.md)

`book-context-map` is a reusable skill for turning a thin book note into a richer context map.

Instead of summarizing a book chapter by chapter, it helps place the book inside a larger knowledge network: who wrote it, what question it is trying to answer, what framework it uses, what should be read before it, what should be read after it, and how it connects to an existing Obsidian vault.

## Why This Exists

When a book is added to a personal knowledge base, the first missing layer is often not "what does this book say?" but "why does this book matter, and where does it belong?"

This skill is designed to fill that gap.

It helps create a context layer that is:

- faster to scan than a long book summary
- more reusable than a one-time reading note
- more connected than an isolated book card

## What The Skill Produces

The skill builds a structured "book context" section that can include:

- author background
- writing context and historical setting
- the book's central question
- theoretical, methodological, or narrative framework
- core concepts
- likely readers and use cases
- prerequisite reading
- further reading
- contrasting or critical perspectives
- links to related notes already in the vault

## Best Fit

This skill works especially well for:

- nonfiction books
- theory and philosophy
- business and methods
- psychology and sociology
- history, biography, and intellectual history
- readers building long-term knowledge networks in Obsidian

It can also be adapted for fiction and poetry by shifting the framework section toward themes, narrative structure, literary tradition, character relationships, and historical context.

## What It Tries To Avoid

This skill is intentionally not optimized for:

- marketing-style book praise
- encyclopedia-style author biography dumps
- empty summaries that just rewrite the table of contents
- fake certainty when the context is incomplete

## Output Style

The default writing style is:

- clear
- restrained
- structured
- retrieval-oriented

The goal is to help the reader quickly understand:

- who this book is for
- what problem it addresses
- what background is useful before reading it
- where to go next after finishing it
- how it connects to other notes and themes
